# 🔍 Анализ результатов тестовой сессии

## 1. Разведочный анализ данных (EDA)

### 1.1. Описание структуры данных

В таблице содержатся следующие столбцы:

- **`session_id`** — ID сессии.  
  Во всех строках значение `1430`, что говорит о том, что все данные относятся к одной и той же тестовой сессии.

- **`id` (первый)** — вероятно, ID пользователя (`user_id`), так как он группирует несколько попыток.  
  Всего **4 уникальных пользователя**: `16904`, `16873`, `16896`, `16897`.

- **`id` (второй)** — вероятно, ID попытки (`submission_id`), уникален для каждой строки.

- **Столбец без названия** (например, `21680`, `21681`…) — скорее всего, ID задания (`task_id`).  
  Каждому номеру задания (1–8) соответствует свой уникальный ID.

- **`№ задания`** — порядковый номер задания в тесте (от 1 до 8).

- **`Балл`** — статус ответа.  
  Возможные значения:
  - `0 none` — попытка не предпринята
  - `0 wrong` — попытка неверная  

  > ⚠️ В данных нет ни одной правильно решённой задачи.

- **`Ответ`** — содержимое ответа.  
  Может быть `NULL` (ответ не дан) или JSON-строкой с деталями решения (текст, код, хэш файла).

- **`Ответ (файлы)`** — ссылка на файл с решением.  
  Часто пустая или является частью данных из столбца `Ответ`.

- **`Фамилия, Имя, Отчество`** — полностью отсутствуют (`#N/A`), для анализа бесполезны.

### 1.2. Качество данных

- **Пропуски**  
  Столбцы ФИО полностью пустые.  
  Столбцы `Ответ` и `Ответ (файлы)` содержат `NULL` или `#N/A`, что является значимой информацией (отсутствие попытки).

- **Неоднородность**  
  Столбец `Ответ` содержит данные разного типа: `NULL` и сложные JSON-структуры (текст, код на Python, ссылки на вложения).  
  Это требует **парсинга** для детального анализа.
